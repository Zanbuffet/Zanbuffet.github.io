NetworkStream和WebSocket对象并不是同一类型的对象，不能直接进行转换。NetworkStream是基于TCP协议的网络流对象，而WebSocket是一种基于HTTP协议的双向通信协议。它们的实现方式不同，用途也不同。

如果你的客户端和服务端都使用了WebSocket协议进行通信，那么在发送消息时应该使用WebSocket对象的SendAsync方法，而不是NetworkStream的WriteAsync方法。如果你的服务端使用了Socket协议而客户端使用了WebSocket协议，你可以在服务端使用WebSocketFactory类将Socket对象转换为WebSocket对象，然后使用WebSocket对象的SendAsync方法发送消息。具体实现方式可以参考之前我提供的代码示例。

依赖注入（Dependency Injection）又叫控制反转(Inversion of Control)
当 class A 使用 class B 的某些功能时，则表示 class A 具有 class B 依赖。
例如在 Java 中，在使用其他 class 的方法之前，我们首先需要创建那个 class 的对象（即 class A 需要创建一个 class B 实例）。
因此，将创建对象的任务转移给其他 class，并直接使用依赖项的过程，被称为“依赖项注入”。
